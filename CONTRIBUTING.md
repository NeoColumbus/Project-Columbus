# Contributing

<!-- HUMAN NOTE: These rules should become stricter as real people join. Do not let openness become a dumping ground for weak language or fake authority. -->

Full City Columbus wants contributions with heat and discipline.

Bring slogans, flyers, neighborhood modules, policy notes, flag concepts, transit propaganda, public comment templates, research corrections, visual identity iterations, memes, and field notes from Columbus streets.

Do not bring beige.

This is an open-source project meant to inspire. You do not need credentials to help. You need a real contribution, a source when you make a factual claim, and enough respect for the city to avoid fake authority.

## New Here?

Start small.

Good first contributions:

- Fix one weak line.
- Add one sourced research correction.
- Make one bus-stop flyer.
- Add one neighborhood-specific detail.
- Replace vague copy with actual Columbus streets, routes, buildings, and pressures.
- Improve image credits or source notes.
- Open an issue with a clear idea someone else can build.

## The Rules

1. No consultant language.
2. No Native cosplay.
3. No fake official affiliation.
4. No OSU color dependency.
5. No beige apartment-core.
6. No policy without beauty.
7. No beauty without material life.
8. AI can scaffold, humans must correct.
9. Make it so hard people want to steal it.
10. Better to fail as myth than succeed as oatmeal.

## How To Submit

Use the templates in [submissions](submissions).

Keep the work specific. A flyer for a bus stop should feel different from a manifesto paragraph. A neighborhood module should name actual streets, routes, buildings, missing crossings, and local beauty.

If you make a historical claim, cite it.
If you use AI, say so.
If you borrow an image, prove that it can be used.
If you speak about a living community, make sure someone from that community can answer back.

## Git Flow

The basic path:

```sh
git checkout -b your-name/one-good-change
git add .
git commit -m "Add one good change"
git push origin your-name/one-good-change
```

Then open a pull request.

If Git still feels weird, that is fine. Open an issue first. Describe the change you want and include the file, line, source, sketch, or rough copy.

Open source is not magic. It is just people leaving a trail other people can build from.

## Tone Test

Before submitting, read the piece out loud.

If it sounds like a grant packet, rewrite it.
If it sounds like a real person could chant it, print it, wear it, argue with it, or steal it, keep going.

## Research Corrections

Corrections are holy work.

Do not be timid. If a file gets history wrong, fix it. If a line overreaches, mark it. If a source is weak, replace it. If a phrase disrespects living people, remove it.

Reverence without accuracy is decoration.
