## Public Arc

Old ground.
Scattered city.
Signal appears.
Body moves.
Proof comes back.

That is the campaign.

Do not make the first touch an essay.
Make it a flag, a wall, a sticker, a route, a photo, a report, a line.

## Roles

Signal Keeper:
posts the mark, carries the flag, keeps the visual language alive.

Wall Reporter:
names blank walls, dead frontages, dead lots, and places where density needs beauty.

Route Scout:
scores bus stops, crossings, sidewalks, shelters, and the routes that make the city move.

Neighborhood Witness:
turns one real place into a line, a flyer, a source note, or a field report.

Machine Tester:
asks what water, power, land, tax, compute, and data the machine owes back.

Source Keeper:
checks history, image rights, claims, dates, and public documents.

Builder:
turns the raw material into pages, posters, templates, bundles, maps, and tools.

## Rituals

Drop Day:
release one symbol, one line, one demand, one public object, and one scan path.

Signal Walk:
take the mark into one corridor and bring back photos, notes, and missing pieces.

Dead Wall Report:
find one wall or frontage where the city goes silent. Name it. Photograph it. Demand life.

Bus Stop Audit:
score one stop for shelter, shade, light, crossing, seating, trash, route information, and dignity.

Data Center Public Test:
run one project through water share, power share, land share, tax share, compute share, data share.

Neighborhood Line:
write one true sentence:

`[PLACE] is not a fragment. It has [ASSET]. It needs [MISSING PIECE]. Build the body.`

## Drop Loop

Every drop needs:

1. Symbol
2. Line
3. Demand
4. Object
5. Scan path
6. Source trail
7. Proof wall

If a drop has no object, it is a thought.
If a drop has no scan path, it is a dead end.
If a drop has no proof wall, it never touched the city.

## Scan Path

Every QR should send people to one action fast.

Drop 001 QR target:

https://neocolumbus.github.io/Project-Columbus/signal/

The scan path should let a person act before they understand GitHub.

Order:

1. choose a door
2. make a field card
3. copy, share, or post the proof
4. send the proof upstream when it is ready
5. publish checked proof on the proof wall

Good scan doors:

- Report a dead wall.
- Score a bus stop.
- Name a neighborhood fragment.
- Test a data center deal.
- Submit a photo from the street.

Bad scan doors:

- Read a theory first.
- Choose from twenty links.
- Guess what the project wants.
- Admire the branding and leave.
