## Git In Plain English

Git is a history machine.

It lets people change files without losing the trail.

The usual open-source move:

1. Fork the project on GitHub.
2. Make a branch.
3. Change files.
4. Commit the change.
5. Open a pull request.

Basic commands:

```sh
git clone YOUR_REPO_URL
cd full-city-columbus
git checkout -b your-name/one-good-change
git add .
git commit -m "Add one good change"
git push origin your-name/one-good-change
```

Then open a pull request on GitHub.
